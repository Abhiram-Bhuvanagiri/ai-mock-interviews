## üõ† Setup ‚Äî run locally (copy & paste)

1. Clone the repo

```bash
git clone https://github.com/Abhiram-Bhuvanagiri/ai-mock-interviews.git
cd ai-mock-interviews
```

2. Install dependencies

```bash
npm install
```

3. Create `.env.local` (see template below). **Do not commit** this file.

```env
NEXT_PUBLIC_VAPI_WEB_TOKEN=pk_xxx
NEXT_PUBLIC_VAPI_WORKFLOW_ID=wf_xxx

NEXT_PUBLIC_FIREBASE_API_KEY=AIza...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your-app.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your-project-id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your-bucket.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=12345
NEXT_PUBLIC_FIREBASE_APP_ID=1:1234:web:abcd

FIREBASE_PROJECT_ID=your-project-id
FIREBASE_CLIENT_EMAIL=your-service-account@your-project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nYOUR_KEY\n-----END PRIVATE KEY-----\n"

GOOGLE_GENERATIVE_AI_API_KEY=
NEXT_PUBLIC_BASE_URL=http://localhost:3000
```

4. Run locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

## ‚úÖ Quick troubleshooting

- If you get `Cannot use "undefined" as a Firestore value` ‚Üí make sure `userId` is passed (guard queries) and `.env.local` keys exist.
- If Firestore returns `FAILED_PRECONDITION: index required` ‚Üí open the Firebase Console > Firestore > Indexes and create the composite index (link appears in the error).
- If sign-in fails with `auth/operation-not-allowed` ‚Üí enable Email/Password (or provider) in Firebase Console > Authentication > Sign-in method and add your domain to Authorized Domains.
- For `FIREBASE_PRIVATE_KEY` formatting in Vercel: replace newlines with `\n` and paste single-line value.

## üì¶ Deploy to Vercel (step-by-step)

1. Go to [https://vercel.com](https://vercel.com) and sign in with GitHub.
2. Click **New Project** ‚Üí import `ai-mock-interviews` repo.
3. Before deploying, add Environment Variables (in the Import flow or Settings) ‚Äî copy all keys from `.env.local` (use `NEXT_PUBLIC_...` for public vars). For `FIREBASE_PRIVATE_KEY` convert newlines to `\n`.
4. Click **Deploy**.
5. After the first deployment, copy the production URL and update `NEXT_PUBLIC_BASE_URL` in Vercel envs and **Redeploy**.
